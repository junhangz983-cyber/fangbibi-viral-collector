# 第三方导出导入

## 推荐输入

日常采集由第三方软件完成，用户交付 CSV 或 Excel：

- `关键词搜索/<关键词>/视频数据.xlsx`
- `对标账号/<账号名>/视频数据.xlsx`
- 多个关键词合并时，用一个批次名，例如 `妈妈7个关键词爆款合计`，不强行给每条记录分配不确定的关键词。

## 社媒助手抖音视频表

当前已验证的列：`视频ID`、`视频链接`、`点赞量`、`评论量`、`分享量`、`发布时间`、`视频时长`、`达人昵称`。

这份导出不含标题、文案或开头钩子。入库时：

- 标题写 `待补标题｜达人昵称｜视频ID`，确保可追溯但不制造假标题。
- 爆点判断只记录互动数据，不能做内容层面的爆点结论。
- 将原帖链接作为唯一去重键。

## 运行示例

```bash
python scripts/import_social_media_export.py \
  --input "/path/to/视频数据.xlsx" \
  --base-token "<base_token>" \
  --table-id "<table_id>" \
  --min-likes 500
```

先增加 `--dry-run` 核验命中数量；写入后必须回读记录数和至少一条样本。
