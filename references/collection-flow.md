# 采集与质量流程

## 两种入口

| 入口 | 输入 | 结果 |
| --- | --- | --- |
| 关键词监测 | 第三方软件按关键词导出的 Excel/CSV | 每个词的高赞候选，合并重复链接 |
| 对标账号 | 第三方软件按账号导出的 Excel/CSV | 账号内容中达到门槛的候选 |

## 关键词监测

1. 导入第三方软件按关键词导出的文件，并记录该文件对应的关键词或合并批次名。
2. 读取标题、账号、互动数据、发布时间、视频 ID 与原帖链接。
3. 过滤点赞低于当前门槛、无链接、标题明显不相关的记录。
4. 同链接合并关键词，写一次飞书。

## 近期要求

搜索排序不等于近期。当用户说“近期”“今日热点”“近 30 天”时：

- 必须读搜索卡片中的发布时间。
- 不能确认时间窗口时，标为“待核验”，不要入近期库。
- 不能用旧内容凑数；结果不足时报告不足。

## 写入前检查

- 点赞的“万”已正确换算为整数。
- 有标题时，标题与关键词至少存在直接语义关联；导出未提供标题时必须标记待补，不能假写。
- 链接未在 Base 中存在。
- `入库日期` 使用本次运行日期；`搜索关键词` 保留全部命中词。
